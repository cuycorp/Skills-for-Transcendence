# File Organization
## src/ app code
### root of directory
+ main.tsx: take the react app and inject in index.html
+ App.tsx: sets up the big picture/main scheme/page layout of the app, sets:
    + page layout
    + routing
    + pull up other components
+ App.css: styles for app
+ index.css: global styles 
define the scope of each one
+ vite-env.d.ts: A tiny TypeScript declaration file that tells TypeScript about Vite-specific features (like importing images). D'ont edit.
### assets
### components
+ resusable components

## outside src/ configuration for vite typescript and npm
### in the root of directory
+ package.json: like Makefile
+ package-lock.json: Auto-generated by npm. Records the exact versions of every installed library 
+ vite.config.ts: tells vite how to build the project: plugins, port, settings, aliases
+ tsconfig.json and tsconfig.node.json:ell the TypeScript compiler how strict to be, which files to include, which modern features to support. Two files because your app code (browser) and your config code (Node.js/Vite) run in different environments and need slightly different settings.(not touched when starting out) 
+ index.html: the entire file lives in it.
    + The <div id="root"> is an empty container — React will inject your entire app into this div
    + The <script> tag points to main.tsx, which is where everything starts
### public/
+ files inside this folder are served as is, e.g. image or logo.
